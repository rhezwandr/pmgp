## Prinsip umum LKM

- LKM bukan tes.
- LKM tidak memiliki skor.
- LKM tidak memiliki benar/salah.
- LKM tidak memakai kunci jawaban otomatis.
- LKM berisi aktivitas belajar, pengalaman belajar, refleksi, dan jawaban mahasiswa.
- Dosen memberi feedback per LKM.

## Struktur yang diharapkan

Setiap LKM sebaiknya memiliki:

- nomor LKM,
- judul/pokok bahasan,
- waktu,
- petunjuk,
- bagian Concrete,
- bagian Pictorial,
- bagian Abstract,
- pertanyaan/perintah,
- kolom jawaban mahasiswa,
- kolom pengalaman belajar/refleksi,
- feedback dosen.

## UI mahasiswa

Mahasiswa dapat:
- membuka LKM 1-6,
- membaca instruksi,
- mengisi jawaban,
- mengisi refleksi/pengalaman belajar,
- menyimpan draft jika tersedia,
- submit.

## UI dosen

Dosen dapat:
- memilih mahasiswa,
- melihat status LKM 1-6,
- membaca jawaban/refleksi mahasiswa,
- memberi feedback,
- mengedit feedback.

## Badge yang disarankan

- Belum dikerjakan
- Draft
- Sudah submit
- Feedback belum diberikan
- Feedback sudah diberikan
